To start the first part, we need two definitions: 
Value is data stored in memory. (textbook, chapter 10)
An Object is a space in memory  that holds information ('memory address', 2017).

 A value can be stored in an object, and it has a Type. It can be either integer, float-point, string, character, boolean, a list can hold more of one type of value, like strings, integers, and even other lists.

Before explaining the results, which may be not intuitive, I will define 2 more terms:
Reference is when a variable points to an object in memory, in the case before the variable a references the object [1 , 2].
When more than one variable references the same object, as we will see in the next image, we say that this object is aliased (textbook, chapter 10.11).



Is operator verifies if the objects have the same ram address. See more about it in references or textbook figure 10.4 in chapter 10.

In the second image, the object [1 , 2] is referenced by the variables 'a' and 'b'. If an object is referenced by more than one variable we say it is aliased. The relation that establishes the aliasing property is 'a == b', in another word, b receive the reference of the same memory address as a ('memory address', 2017), so the objects are identical.
